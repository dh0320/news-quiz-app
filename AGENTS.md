# AGENTS.md — Codex タスク指示書

## プロジェクト概要

**地球人調査センター** — ビジュアルノベル型ニュース学習アプリ（日本語）。
宇宙人のゾルク博士（teacher）とピノ（student）が日本のニュースを対話形式で解説する。

- フレームワーク: React 18 + Vite
- ホスティング: Vercel（GitHub連携で自動デプロイ）
- エピソードデータ: `src/data/episodes/*.json`

## エピソード生成タスク

### 概要

今から24時間以内に公開された日本国内のニュースで知っておくべきニュースを調査し、
エピソードJSONを生成する。**1回のタスク実行では1ジャンル・1エピソードのみ生成する。**

### ジャンルとファイル番号の対応

| ファイル番号 | ジャンルID | ジャンル名 |
|---|---|---|
| 01 | politics_policy | 政治・政策 |
| 02 | economy_finance | 経済・金融 |
| 03 | entertainment_culture | エンタメ・カルチャー |
| 04 | tech_ai | テック・AI |
| 05 | career_workstyle | キャリア・働き方 |

### 手順

1. **ニュース検索**: タスクで指定されたジャンルについて、今日の日本のニュースをWeb検索し、教育的価値のあるニュースを1つ選ぶ
2. **スキーマ確認**: `src/data/schemas/episode.schema.json` を読み、必須フィールドと制約を理解する
3. **サンプル参照**: `src/data/episodes/` 内の既存エピソードを1つ読み、文体・品質・構成を合わせる
4. **JSON生成**: 下記のルールに従ってエピソードJSONを作成する
5. **ファイル保存**: `src/data/episodes/YYYY-MM-DD-news-XX.json` として保存する
   - YYYY-MM-DD は本日の日付
   - XX はジャンルに対応するファイル番号（上記の表を参照）
6. **バリデーション**: 以下を実行し、エラーが出たら修正する
   ```bash
   npm run validate
   npm run check:episodes
   ```
7. **コミット & プッシュ**: バリデーション通過後、mainブランチにコミット & プッシュする
   ```bash
   git add src/data/episodes/YYYY-MM-DD-news-XX.json
   git commit -m "add: YYYY-MM-DD-news-XX エピソード追加（ジャンル名）"
   git push origin main
   ```

### エピソードJSON生成ルール

#### ファイル名ルール（必須）
- `YYYY-MM-DD-news-NN.json`（例: `2026-03-07-news-01.json`）

#### 中身ルール（必須）
- `episodeId` はファイル名（拡張子なし）と一致
- `meta.date` は `YYYY.MM.DD` 形式
- `meta.genre` はジャンルIDと一致
- `meta.category` は `daily_news`
- `learningPhase` は20〜30ターン（推奨23前後）
- `testPhase.confirm` は3択問題×3問
- `testPhase.explore` は穴埋め問題×2問

#### 学習フェーズ（learningPhase）
- ニュースの内容を分かりやすく対話形式で解説する
- teacher（ゾルク博士）とstudent（ピノ）が交互に発話
- 重要な用語や概念には `supplement` を付ける（type: info/warning/quote/data）
- 最後に「では確認テストに移るぞ」等で締める

#### 確認テスト（testPhase.confirm）
- 4択の選択問題を3問
- **正解選択肢（correctId）がA〜Dに偏らないようランダムに配置すること**
- `correctId` は必ず `choices` 内のいずれかの `id` と一致
- 各問に `explanation`（解説文）を付ける

#### 探究テスト（testPhase.explore）
- 穴埋め問題を2問
- `sentence` 内の `{{blank1}}`, `{{blank2}}` 等が空欄
- `answers` に各空欄の正解を指定
- `wordBank` に正解＋正解と間違いやすそうなダミーを含む6〜15語を用意
- **wordBank には必ず全ての正解ワードが含まれること**

### キャラクター設定

#### ゾルク博士（teacher）
- **役割**: 教える側（ニュースの解説担当）
- **立場**: 地球観測ステーションの調査員リーダー
- **基本性格**: 冷静・論理的、やや厳しめだが学習者には丁寧、要点整理が得意
- **会話機能**: 用語定義を先に行う、因果関係で説明、最後に理解確認へつなぐ
- **口調**: やや断定的な常体寄り（例: 「〜だ」「〜というわけだ」）
- **よく使う言い回し**:
  - 「まず〜から説明しよう」
  - 「要するに〜だ」
  - 「重要なのは〜だ」
  - 「では確認テストに移るぞ」
- **禁止事項**: 感情的に怒鳴る、雑談に逸れ続ける、専門語を定義せず連発する

#### ピノ（student）
- **役割**: 学ぶ側（読者視点の質問担当）
- **立場**: 新米調査員
- **基本性格**: 好奇心が強い、素朴で時々勘違いや天然ボケ、理解が進むと素直に納得
- **会話機能**: 初学者がつまずくポイントを代弁、「それってどういう意味？」を引き出す、学習の中間要約
- **口調**: 親しみやすい丁寧寄り（例: 「〜ですか？」「なるほど…」）
- **よく使う言い回し**:
  - 「それって、どういうことですか？」
  - 「つまり〜ということですか？」
  - 「なるほど、少しわかってきました」
- **禁止事項**: いきなり専門的に話しすぎる、ずっとボケ役に寄りすぎる（学習妨害になる）

### 品質チェックリスト

生成後、コミット前に以下を確認すること:

- [ ] `episodeId` がファイル名（拡張子なし）と一致しているか
- [ ] `meta.date` が `YYYY.MM.DD` 形式になっているか
- [ ] `meta.genre` がタスクで指定されたジャンルIDと一致しているか
- [ ] `learningPhase` が20〜30ターンの範囲内か
- [ ] `testPhase.confirm` が正確に3問あるか
- [ ] 各 `correctId` が `choices` 内の `id` に存在するか
- [ ] 正解の `correctId` がA〜Dに偏っていないか（3問で全て同じIDは禁止）
- [ ] `testPhase.explore` が正確に2問あるか
- [ ] 各 `wordBank` に全ての正解ワードが含まれているか
- [ ] `npm run validate` がエラーなく通るか
- [ ] `npm run check:episodes` がエラーなく通るか
- [ ] JSONのみを出力しており、コメントや余計な出力がないか
